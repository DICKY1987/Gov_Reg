## What To Do

Define blocked anti-patterns, operator notes, waiver process, and no-skeleton-code enforcement.

---

## How To Do It

### Step 1: Define Anti-Patterns

**Standard anti-patterns (always block):**

1. **ANTI-GIANT-REFACTOR-001**
   - Pattern: Massive refactors without incremental validation
   - Detection: Diff size > 2000 lines in single PR
   - Mitigation: Break into smaller phases with validation gates

2. **ANTI-EDIT-OUTSIDE-SCOPE-001**
   - Pattern: Editing files outside declared file_scope
   - Detection: File touch not in allowed paths
   - Mitigation: Abort phase, require scope amendment

3. **ANTI-SKIP-TESTS-001**
   - Pattern: Skipping tests without justification
   - Detection: pytest.mark.skip without reason
   - Mitigation: Require waiver with tracking issue

4. **ANTI-HARDCODED-SECRETS-001**
   - Pattern: Hardcoded credentials or API keys
   - Detection: Secrets scanner finds matches
   - Mitigation: Use environment variables or secret store

5. **ANTI-NO-ERROR-HANDLING-001**
   - Pattern: Missing try/except in critical paths
   - Detection: Static analysis finds uncaught exceptions
   - Mitigation: Add error handling with logging
