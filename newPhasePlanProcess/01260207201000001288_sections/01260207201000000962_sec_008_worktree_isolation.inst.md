# SEC-008: Worktree Isolation + Merge Strategy Instructions

**Schema File:** `sections/sec_008_worktree_isolation.json`  
**Section ID:** `sec_008_worktree_isolation`  
**Required:** No (Optional)  
**Updated:** 2026-01-28

---

## What To Do

Configure git worktree isolation for parallel phase execution (optional for simple projects).

---

## How To Do It

### Step 1: Decide If Worktree Isolation Needed

**Enable if:**
- Multiple phases run in parallel
- Phases touch different file sets
- Need isolation to prevent conflicts

**Skip if:**
- Simple/moderate project (sequential phases)
- Single developer, no parallelization

### Step 2: Configure Strategy

```json
{
  "enabled": true,
  "strategy": {
    "mode": "per_workstream",
    "base_branch": "main",
    "max_parallel_workers": 3,
    "naming_patterns": {
      "worktree_dir": ".worktrees/wt-${workstream_id}-${phase_id}",
      "branch_name": "feature/${phase_id}-${workstream_id}"
    }
  }
}
```

### Step 3: Define Merge Strategy

```json
{
  "merge_strategy": {
    "method": "squash-merge",
    "conflict_policy": "abort_and_escalate",
    "pre_merge_validations": [
      "All tests pass in worktree",
      "No scope violations detected"
    ]
  }
}
```

---

## Validation Rules

1. If enabled=true, all strategy fields required
2. max_parallel_workers ≤ CPU cores
3. merge_strategy must be specified

---

## AI Agent Notes

- Most projects can skip this (set enabled=false)
- Useful for enterprise projects with parallel workstreams
- Each worktree is isolated - no cross-contamination
