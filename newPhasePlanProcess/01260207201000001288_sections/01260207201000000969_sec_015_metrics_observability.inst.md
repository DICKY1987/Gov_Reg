## What To Do

Define quantified metrics to track and observability configuration.

---

## How To Do It

### Step 1: Define Tracked Metrics

**Standard metrics (always include):**

1. **MET-001: test_coverage_percentage**
   - Target: 95% (or per classification)
   - Command: `pytest --cov=src --cov-report=json:.runs/coverage.json && jq '.totals.percent_covered' .runs/coverage.json`

2. **MET-002: lint_errors**
   - Target: 0
   - Command: `ruff check src/ --output-format json | jq 'length'`

3. **MET-003: type_coverage_percentage**
   - Target: 90%
   - Command: `mypy src/ --txt-report .runs/mypy-report`

4. **MET-004: build_time_seconds**
   - Target: ≤60
   - Command: `time python -m build`

5. **MET-005: test_execution_time_seconds**
   - Target: ≤300
   - Command: `pytest --durations=0 -q`

6. **MET-006: security_issues_high_critical**
   - Target: 0
   - Command: `bandit -r src/ -f json | jq '[.results[] | select(.issue_severity | IN("HIGH", "CRITICAL"))] | length'`

7. **MET-007: documentation_coverage_percentage**
   - Target: 80%
   - Command: `python scripts/check_doc_coverage.py src/`

8. **MET-008: determinism_score**
   - Target: 0.95
   - Command: `python scripts/test_determinism.py --runs 3`

### Metric Entry Format

```json
{
  "metric_id": "MET-001",
  "metric_name": "test_coverage_percentage",
  "target_value": 95,
  "target_operator": ">=",
  "current_value": null,
  "unit": "percent",
  "collection_command": "{{COMMAND}}"
}
```

### Step 2: Configure Storage

```json
{
  "storage": {
    "format": "jsonl",
    "path": ".state/metrics.jsonl",
    "rotation_policy": "size_based",
    "max_size_mb": 100,
    "keep_files": 10
  }
}
```

### Step 3: Configure Event Logging

```json
{
  "event_logging": {
    "enabled": true,
    "format": "jsonl",
    "path": ".state/events.jsonl",
    "event_tags": [
      "phase:${phase_id}",
      "workstream:${workstream_id}"
    ]
  }
}
```

---

## Validation Rules

1. ≥8 metrics defined
2. Each metric has: metric_id, metric_name, target_value, target_operator, collection_command
3. Storage configuration specified
4. Event logging configured
