# SEC-018: GitHub Integration Instructions

**Schema File:** `sections/sec_018_github_integration.json`  
**Section ID:** `sec_018_github_integration`  
**Required:** No (Optional)  
**Updated:** 2026-01-28

---

## What To Do

Configure optional GitHub integration for issue/project sync (skip if not using GitHub).

---

## How To Do It

### Step 1: Decide If GitHub Integration Needed

**Enable if:**
- Project tracked in GitHub Issues
- Using GitHub Projects for planning
- Want automated phase → issue sync

**Skip if:**
- Not using GitHub
- No need for automated sync
- Tracking elsewhere

### Step 2: Configure Repository

```json
{
  "enabled": true,
  "repository": {
    "owner": "myorg",
    "name": "myrepo",
    "default_branch": "main",
    "visibility": "private"
  }
}
```

### Step 3: Configure Issue Management

```json
{
  "issue_management": {
    "mode": "one-per-phase",
    "issue_config": {
      "title_template": "[{phase_id}] {title}",
      "body_template_path": "templates/github_issue.md",
      "labels": ["phase-plan", "ai-generated"],
      "assignees": ["myusername"]
    }
  }
}
```

### Step 4: Configure Project Sync

```json
{
  "project_management": {
    "url": "https://github.com/orgs/myorg/projects/5",
    "owner": "myorg",
    "project_number": 5,
    "field_mappings": {
      "phase_id_field": "Phase ID",
      "status_field": "Status"
    },
    "status_mappings": {
      "planned": "Todo",
      "in_progress": "In Progress",
      "blocked": "Blocked",
      "done": "Done"
    }
  }
}
```

### Step 5: Configure Automation

```json
{
  "automation": {
    "allow_issue_create": true,
    "allow_issue_update": true,
    "sync_direction": "yaml->github",
    "on_phase_status_change_update_project": true
  }
}
```

---

## Validation Rules

1. If enabled=true, repository config required
2. If issue_management configured, mode must be specified
3. If project_management configured, URL and mappings required
4. Sync script command specified
