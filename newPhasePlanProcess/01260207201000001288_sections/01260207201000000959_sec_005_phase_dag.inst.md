# SEC-005: Phase DAG + Parallel Execution Instructions

**Schema File:** `sections/sec_005_phase_dag.json`  
**Section ID:** `sec_005_phase_dag`  
**Required:** Yes  
**Updated:** 2026-01-28

---

## What To Do

Create a Directed Acyclic Graph (DAG) of all phases with:
1. Phase index (all phases listed)
2. Dependencies between phases
3. Parallel execution groups
4. Time estimates and bottlenecks

---

## How To Do It

### Step 1: List All Phases

Based on classification from SEC-002:
- **Simple:** intake, implement, verify
- **Moderate:** intake, plan, implement, verify
- **Complex:** intake, architecture, implement, integration, verify
- **Enterprise:** bootstrap, architecture, contracts, implement_parallel, integration, hardening, package

### Step 2: Define Dependencies

For each phase, specify:
- `depends_on`: HARD dependencies (must complete first)
- `may_run_parallel_with`: Safe to run concurrently
- `is_critical_path`: Affects total timeline

Example:
```json
{
  "phase_id": "PH-003",
  "depends_on": ["PH-001", "PH-002"],
  "may_run_parallel_with": ["PH-010"],
  "is_critical_path": true
}
```

### Step 3: Create Parallel Groups

Group phases that can run simultaneously:
```json
{
  "group_id": "PG-001",
  "group_name": "Implementation",
  "phases": ["PH-003", "PH-004", "PH-005"],
  "concurrency_limit": 3
}
```

### Step 4: Calculate Time Estimates

- Sequential: Sum all phase durations
- Parallel: Account for concurrent execution
- Speedup: (Sequential - Parallel) / Sequential * 100

---

## Validation Rules

1. No circular dependencies (DAG)
2. All phases have unique IDs
3. Dependencies reference valid phase IDs
4. At least one phase has no dependencies (entry point)
5. At least one phase has no dependents (exit point)

---

## AI Agent Notes

- Critical path determines minimum timeline
- Parallel execution can reduce total time 30-70%
- Bottlenecks are single-threaded phases on critical path
