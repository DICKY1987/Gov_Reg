**Required:** Yes  
**Updated:** 2026-01-28

---

## What To Do

Create a machine-checkable contract for EVERY phase in the plan.

---

## How To Do It

### Phase Contract Structure

Each phase contract MUST have:

1. **phase** - Identity and metadata
2. **dependencies** - Which phases must complete first
3. **scope** - Path policy (allowed/forbidden globs)
4. **tasks** - Atomic work items
5. **artifacts** - Files created/modified/deleted
6. **validations** - Commands that prove correctness
7. **completion_gate** - Rules that must pass
8. **extensions** - Risks, questions, entry criteria

### Step 1: Define Phase Identity

```json
{
  "phase": {
    "phase_id": "PH-001",
    "workstream_id": "WS-001",
    "title": "Bootstrap Environment",
    "summary": "Set up Python environment and install dependencies",
    "objective": "Python 3.11 venv created with all dependencies installed",
    "phase_type": "implementation",
    "status": "planned"
  }
}
```

### Step 2: Define Scope with Path Policy

```json
{
  "scope": {
    "path_policy": {
      "allowed": [{
        "rule_id": "PATH-ALLOW-001",
        "globs": ["requirements.txt", ".venv/**"],
        "modes": ["read", "create", "modify"]
      }],
      "forbidden": [{
        "rule_id": "PATH-DENY-001",
        "globs": ["src/**", "tests/**"],
        "modes": ["create", "modify", "delete"]
      }]
    }
  }
}
```

**CRITICAL:** Every file touch MUST be in `allowed` globs.

### Step 3: Define Tasks

```json
{
  "tasks": [{
    "task_id": "TASK-001",
    "title": "Create virtual environment",
    "description": "Create Python 3.11 venv at .venv/",
    "acceptance_criteria": [
      ".venv/ directory exists",
      "python interpreter is 3.11+"
    ],
    "artifact_ids_touched": ["ART-001"],
    "validation_ids": ["VAL-001"]
  }]
}
```

### Step 4: Define Artifacts

```json
{
  "artifacts": [{
    "artifact_id": "ART-001",
    "path": ".venv/bin/python",
    "action": "create",
    "classification": "config",
    "owner": {"phase_id": "PH-001", "task_id": "TASK-001"},
    "must_exist_after_phase": true
  }]
}
```

**Actions:** read, create, modify, delete

### Step 5: Define Validations

```json
{
  "validations": [{
    "validation_id": "VAL-001",
    "kind": "command",
    "description": "Verify Python 3.11+ installed",
    "command": ".venv/bin/python --version",
    "timeout_seconds": 10,
    "success_criteria": {
      "exit_code": 0,
      "stdout_patterns_all": ["/Python 3\\.1[1-9]/"]
    },
    "evidence_output": {
      "log_path": ".runs/PH-001/val_001.log"
    },
    "must_pass": true
  }]
}
```

### Step 6: Define Completion Gate

```json
{
  "completion_gate": {
    "rules": {
      "dependencies_satisfied": true,
      "all_required_artifacts_present": true,
      "no_scope_violations": true,
      "all_must_pass_validations_passed": true
    }
  }
}
```

---

## Validation Rules

1. Every task references only existing artifact_ids and validation_ids
2. scope.path_policy has ≥1 allowed AND ≥1 forbidden rule
3. All artifacts have: artifact_id, path, action, owner
4. All validations have: command, timeout, success_criteria, evidence_path
5. completion_gate.rules fully specified
