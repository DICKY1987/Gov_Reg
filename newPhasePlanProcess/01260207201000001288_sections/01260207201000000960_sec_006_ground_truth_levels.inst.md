---

## What To Do

Define 6 validation levels (L0-L5) with executable commands and evidence paths.

---

## How To Do It

### Validation Levels

**L0: Syntax + Basic Validation**
- Files parse correctly
- Basic structure valid
- Commands: `python -m py_compile`, `yamllint`

**L1: Static Analysis**
- Lint checks pass
- Type checking passes
- Security scan clean
- Commands: `ruff check`, `mypy`, `bandit`

**L2: Unit Tests**
- All unit tests pass
- Coverage target met (70-95% based on classification)
- Commands: `pytest tests/unit --cov=src`

**L3: Integration Tests**
- Component interactions verified
- Commands: `pytest tests/integration`

**L4: End-to-End Tests**
- Full workflows verified
- Commands: `pytest tests/e2e`

**L5: Production Validation**
- Smoke tests in prod-like environment
- Commands: `docker-compose up`, smoke tests

### For Each Level, Specify:

```json
{
  "level": "L2",
  "name": "Unit Tests",
  "commands": [{
    "command": "pytest tests/unit -v --cov=src",
    "timeout_seconds": 300,
    "expect_exit_code": 0,
    "expect_stdout_regex": ["/PASSED/", "/TOTAL.*9[5-9]|100%/"]
  }],
  "evidence_path": ".runs/validation/L2_unit_tests.log",
  "required_for_phases": ["implement", "refactor"]
}
```

### Map Phases to Minimum Levels

- Intake: L0
- Implement: L2
- Integration: L3
- Hardening/Package: L4-L5

---

## Validation Rules

1. All 6 levels defined (L0-L5)
2. Each level has ≥1 command
3. All commands have timeout and evidence path
4. Per-phase requirements map phases to levels

---

## AI Agent Notes

- Higher levels include lower levels (L3 implies L0, L1, L2 pass)
- Classification determines minimum required level
- Evidence paths must be under `.runs/validation/`
