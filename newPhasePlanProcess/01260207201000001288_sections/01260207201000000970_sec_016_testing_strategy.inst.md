# SEC-016: Testing Strategy Instructions

**Schema File:** `sections/sec_016_testing_strategy.json`  
**Section ID:** `sec_016_testing_strategy`  
**Required:** Yes  
**Updated:** 2026-01-28

---

## What To Do

Define test-first development approach with test layers, fixtures, and quality gates.

---

## How To Do It

### Step 1: Enable Test-First Development

```json
{
  "test_first_development": {
    "enabled": true,
    "description": "Write tests alongside code (TDD)",
    "requirement": "Tests must pass 100% (or explicit exception with mitigation)"
  }
}
```

### Step 2: Define Test Layers

**Unit Tests:**
```json
{
  "unit": {
    "description": "Test individual functions/classes in isolation",
    "coverage_target": "95%",
    "location": "tests/unit/**",
    "run_command": "pytest tests/unit -v --tb=short --cov=src",
    "timeout_seconds": 300,
    "quality_gates": [
      "0 failures",
      "Coverage >= target",
      "No skipped tests without justification"
    ]
  }
}
```

**Integration Tests:**
```json
{
  "integration": {
    "description": "Test component interactions",
    "coverage_target": "80%",
    "location": "tests/integration/**",
    "run_command": "pytest tests/integration -v --tb=short",
    "timeout_seconds": 600,
    "quality_gates": [
      "0 failures",
      "All critical paths tested"
    ]
  }
}
```

**E2E Tests:**
```json
{
  "e2e": {
    "description": "Test complete workflows end-to-end",
    "coverage_target": "70%",
    "location": "tests/e2e/**",
    "run_command": "pytest tests/e2e -v --tb=short",
    "timeout_seconds": 1800,
    "quality_gates": [
      "0 failures",
      "All user stories covered"
    ]
  }
}
```

**Determinism Tests:**
```json
{
  "determinism": {
    "description": "Verify outputs are stable across runs",
    "location": "tests/determinism/**",
    "run_command": "python scripts/test_determinism.py --runs 3",
    "timeout_seconds": 600,
    "quality_gates": [
      "Determinism score >= 0.95"
    ]
  }
}
```

### Step 3: Define Test Fixtures

```json
{
  "test_fixtures": {
    "valid_inputs": {
      "location": "tests/fixtures/valid/**",
      "description": "Sample valid input data",
      "count": 10
    },
    "invalid_inputs": {
      "location": "tests/fixtures/invalid/**",
      "description": "Sample invalid input data with expected errors",
      "count": 10
    },
    "expected_outputs": {
      "location": "tests/fixtures/expected/**",
      "description": "Expected outputs for each input",
      "format": "json"
    }
  }
}
```

### Step 4: Set Quality Gates

```json
{
  "quality_gates_summary": {
    "run_all_tests_command": "pytest tests/ -v --tb=short --cov=src",
    "expect_exit_code": 0,
    "gates": [
      "100% of tests pass",
      "Coverage >= classification-based target",
      "Determinism score >= 0.95"
    ]
  }
}
```

---

## Validation Rules

1. All 4 test layers defined (unit, integration, e2e, determinism)
2. Each layer has: location, run_command, timeout, quality_gates
3. Test fixtures defined for valid/invalid inputs
4. Quality gates summary specified

---

## AI Agent Notes

- Tests are written WITH code, not after
- Coverage targets based on classification (simple=70%, enterprise=95%)
- Determinism tests verify reproducibility
- 100% pass rate is non-negotiable
