# SEC-014: Decision Ledger Instructions

**Schema File:** `sections/sec_014_decision_ledger.json`  
**Section ID:** `sec_014_decision_ledger`  
**Required:** Yes  
**Updated:** 2026-01-28

---

## What To Do

Create append-only log of ALL decisions made during planning.

---

## How To Do It

### For Each Decision, Record:

```json
{
  "decision_id": "DEC-PROJECT-001",
  "category": "tooling",
  "timestamp": "2026-01-28T06:09:17.948Z",
  "context": {
    "problem": "Need JSON schema validation library",
    "constraints": ["Must support draft-07 and draft-2020-12", "Active maintenance"]
  },
  "options": ["jsonschema", "pydantic", "custom implementation"],
  "selected_option": "jsonschema",
  "rationale": "jsonschema library has best draft-2020-12 support, actively maintained, 50M+ downloads/month",
  "owner": "AI_AGENT",
  "impact": "MEDIUM",
  "reversible": "YES",
  "validation": {
    "command": "pip show jsonschema | grep Version",
    "expect_exit_code": 0
  }
}
```

### Decision Categories

- **classification** - Project complexity level
- **architecture** - System design choices
- **tooling** - Tool/library selections
- **process** - Workflow decisions
- **technical** - Implementation choices
- **scope** - In/out of scope decisions
- **quality** - Quality gate selections

### When to Log a Decision

Log EVERY decision where you chose between alternatives:
- Library/tool selection
- Design pattern choice
- File structure
- Testing approach
- Validation method
- Phase ordering

**Don't log:**
- Obvious/default choices with no alternatives
- Mechanical derivations from other data

**Target:** 10-20 decisions for a moderate project

---

## Validation Rules

1. ≥5 decisions logged
2. Each has: decision_id, category, options (≥2), selected_option, rationale
3. All decision_ids unique and follow pattern: DEC-{PROJECT}-{NNN}
4. All timestamps in ISO 8601 format
5. Validation command defined if decision is verifiable

---

## Canonical Storage Path

The ledger JSONL file resolves to **`.state/evidence/decisions/decision_log.jsonl`**. This is the path declared in `NEWPHASEPLANPROCESS_AUTONOMOUS_DELIVERY_TEMPLATE_V3_3.json` (`decision_logging_policy.log_targets[0]`); the technical spec registers `decision_logging_policy` as the governing mechanism. SEC-014 and aggregate section definitions must use this template log target.
