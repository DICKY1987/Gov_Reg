# SEC-011: Artifact Manifest Instructions

**Schema File:** `sections/sec_011_artifact_manifest.json`  
**Section ID:** `sec_011_artifact_manifest`  
**Required:** Yes  
**Updated:** 2026-01-28

---

## What To Do

Create cross-phase "bill of materials" for ALL files touched across the plan.

---

## How To Do It

### Step 1: Aggregate Artifacts from Phase Contracts

Extract ALL artifacts from SEC-010 phase contracts and consolidate into one table.

### Step 2: Create Manifest Table

For each artifact:
```json
{
  "phase_id": "PH-001",
  "artifact_id": "ART-001",
  "action": "create",
  "path": "src/pipeline/state.py",
  "introduced_in_phase": "PH-001",
  "purpose": "State machine implementation",
  "validation_ids": ["VAL-001", "VAL-002"],
  "validation_command_hint": "pytest tests/test_state.py"
}
```

### Step 3: Verify Consistency

Run consistency check:
```bash
python scripts/validate_artifact_manifest.py {{PLAN_FILE}}
```

**Checks:**
- All artifact_ids from phase contracts appear in manifest
- All paths are unique within action type
- No artifacts reference non-existent validation_ids
- introduced_in_phase matches first appearance

---

## Validation Rules

1. manifest_table.entries has ALL artifacts from ALL phase contracts
2. Each entry has: phase_id, artifact_id, action, path, purpose
3. No duplicate artifact_ids
4. All validation_ids reference valid validations
5. Consistency check command defined
