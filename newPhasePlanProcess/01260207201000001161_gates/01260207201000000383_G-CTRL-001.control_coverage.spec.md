# G-CTRL-001: Control Coverage Gate Specification

**Gate ID**: G-CTRL-001  
**Version**: 1.0.0  
**Purpose**: Enforce that all required quality controls have complete mappings (gates + evidence + artifacts + tests)  
**Last Updated**: 2026-01-28T17:04:10.413Z

---

## Algorithm

### Input

- **SEC-021**: Quality controls catalog (`sections/sec_021_quality_controls_catalog.json`)
- **SEC-012**: Validation gates (`sections/sec_012_validation_gates.json`)
- **SEC-011**: Artifact manifest (`sections/sec_011_artifact_manifest.json`)
- **SEC-016**: Testing strategy (`sections/sec_016_testing_strategy.json`)
- **Evidence Policy**: Deterministic path formulas (`policies/evidence_path_policy.json`)

### Output

- **Exit Code**: 0 if all required controls satisfied, 1 if violations found, 2 if internal error
- **Report**: JSON file at `evidence/{plan_id}/{run_id}/gates/G-CTRL-001/report.json`
- **Stdout**: Machine-readable summary with `all_required_controls_satisfied: true|false`

---

## Algorithm Steps

### Step 1: Load SEC-021 Controls

```
1. Load sections/sec_021_quality_controls_catalog.json
2. Extract quality_controls_catalog.controls
3. Compute RequiredControls = [c for c in controls if c.required == true]
4. Sort RequiredControls by control_id (lexicographic, stable ordering)
```

### Step 2: Load SEC-012 Gates

```
1. Load sections/sec_012_validation_gates.json
2. Extract validation_gates.gates array
3. Build ControlToGates map:
   - For each gate in gates:
     - If gate has 'controls_satisfied' field:
       - For each control_id in controls_satisfied:
         - ControlToGates[control_id].append(gate)
4. Sort all arrays in ControlToGates by gate_id
```

### Step 3: Validate Each Required Control

For each control in RequiredControls (sorted):

#### Check (a): Satisfying Gate Exists

```
IF control.control_id NOT IN ControlToGates:
    EMIT ERROR: CTRL001_MISSING_SATISFYING_GATE
    - control_id: <control_id>
    - message: "Control marked required=true but no gate declares controls_satisfied: [<control_id>]"
```

#### Check (b): Evidence Mapping Exists

```
FOR each gate in ControlToGates[control.control_id]:
    IF gate does NOT have 'evidence_outputs' field:
        EMIT ERROR: CTRL001_MISSING_EVIDENCE_MAPPING
        - control_id: <control_id>
        - gate_id: <gate.gate_id>
        - message: "Gate claims to satisfy control but has no evidence_outputs"
    
    ELSE:
        FOR each evidence_path in gate.evidence_outputs:
            IF evidence_path does NOT match evidence_path_policy formula pattern:
                EMIT ERROR: CTRL001_NONDETERMINISTIC_PATH
                - control_id: <control_id>
                - gate_id: <gate.gate_id>
                - evidence_path: <evidence_path>
                - message: "Evidence path does not use deterministic formula from evidence_path_policy.json"
```

#### Check (c): Required Artifacts Resolve

```
IF control has 'required_artifacts' field AND it is non-empty:
    Load sections/sec_011_artifact_manifest.json
    Extract artifact_manifest.manifest_table.entries
    
    FOR each artifact_id in control.required_artifacts:
        IF artifact_id NOT IN [e.artifact_id for e in entries]:
            EMIT ERROR: CTRL001_MISSING_REQUIRED_ARTIFACT
            - control_id: <control_id>
            - artifact_id: <artifact_id>
            - message: "Control requires artifact but it is not declared in SEC-011 artifact manifest"
```

#### Check (d): Required Tests Resolve

```
IF control has 'test_requirements' field:
    Load sections/sec_016_testing_strategy.json
    
    IF sec_016 has 'control_test_mapping' section:
        mappings = sec_016.control_test_mapping.mappings
        
        IF control.control_id NOT IN [m.control_id for m in mappings]:
            EMIT ERROR: CTRL001_MISSING_REQUIRED_TEST
            - control_id: <control_id>
            - message: "Control specifies test_requirements but no mapping exists in SEC-016"
```

### Step 4: Generate Report

```
1. Compute satisfied_controls = [c for c in RequiredControls if no errors emitted for c]
2. Compute violated_controls = [c for c in RequiredControls if errors emitted for c]
3. Sort all arrays by control_id

4. Generate report.json:
   {
     "gate_id": "G-CTRL-001",
     "status": "pass" | "fail",
     "start_timestamp": "<ISO8601>",
     "end_timestamp": "<ISO8601>",
     "duration_sec": <float>,
     "required_controls_count": <int>,
     "satisfied_controls_count": <int>,
     "violated_controls_count": <int>,
     "satisfied_controls": [<sorted_control_ids>],
     "violated_controls": [<sorted_control_ids>],
     "errors": [<sorted_error_objects>],
     "all_required_controls_satisfied": <boolean>
   }

5. Write report to evidence/{plan_id}/{run_id}/gates/G-CTRL-001/report.json

6. Print to stdout:
   all_required_controls_satisfied: <true|false>
   satisfied_controls_count: <int>
   violated_controls_count: <int>

7. Exit with:
   - exit_code=0 if all_required_controls_satisfied == true
   - exit_code=1 if all_required_controls_satisfied == false
```

## Error Codes

| Code | Meaning | Remediation |
|------|---------|-------------|
| **CTRL001_MISSING_SATISFYING_GATE** | Control marked `required=true` but no gate in SEC-012 declares `controls_satisfied: [<control_id>]` | Add gate to SEC-012 that satisfies this control, OR mark control `required=false` |
| **CTRL001_MISSING_EVIDENCE_MAPPING** | Gate claims to satisfy control but has no `evidence_outputs` field | Add `evidence_outputs: [...]` to gate in SEC-012 |
| **CTRL001_NONDETERMINISTIC_PATH** | Evidence path does not use deterministic formula from `evidence_path_policy.json` | Update evidence path to use formula pattern (e.g., `evidence/{plan_id}/{run_id}/gates/{gate_id}/`) |
| **CTRL001_MISSING_REQUIRED_ARTIFACT** | Control requires artifact but it is not declared in SEC-011 manifest | Add artifact entry to SEC-011, OR remove from `required_artifacts` |
| **CTRL001_MISSING_REQUIRED_TEST** | Control specifies `test_requirements` but no mapping exists in SEC-016 | Add control-test mapping to SEC-016 `control_test_mapping` section |

## Determinism Guarantees

1. **Stable Ordering**: All arrays sorted by ID before iteration/output
2. **Idempotent**: Same inputs always produce identical report.json (except timestamps)
3. **No Side Effects**: Read-only operations, no mutations
4. **Deterministic Errors**: Error codes and messages are pre-defined
5. **Path Validation**: Evidence paths checked against policy formulas
