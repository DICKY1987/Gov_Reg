# Implementation Plan: Capability Mapping System

## Context

This plan implements a three-step system to create a comprehensive, evidence-based mapping between repository files and their capabilities. The system will:

1. **Discover capabilities** from executable code (CLI commands, gate validators), schemas, and documentation
2. **Build a complete file inventory** with objective metadata (imports, exports, subprocess calls, path references)
3. **Create a purpose registry** that binds files to capabilities with full traceability

This mapping enables:
- **Gate enforcement** - ensure all critical files are accounted for
- **Drift prevention** - detect when files change purpose
- **Dependency tracking** - understand file relationships
- **Documentation generation** - auto-generate capability documentation

The implementation follows strict determinism requirements (reproducible outputs across runs) and uses a precedence-based conflict resolution system when sources disagree about capabilities.

## Step 1: Capability Discovery

**Module:** `capability_discoverer.py`

### Objective
Scan the repository and discover all capabilities from:
- **CLI commands** (argparse subcommands)
- **Gate validators** (scripts with `GATE-###` pattern)
- **Schemas** (JSON files with `$id` fields)
- **Templates/specs** (enforcement-backed features)

### Key Classes

```python
class CapabilityDiscoverer:
    def discover_all(self, repo_root: Path) -> Dict
    def discover_from_cli(self, scripts_dir: Path) -> List[Capability]
    def discover_from_gates(self, repo_root: Path) -> List[Capability]
    def discover_from_schemas(self, schemas_dir: Path) -> List[Capability]
    def deduplicate_and_resolve_conflicts(capabilities) -> Tuple[Dict, List[Dict]]
```

### Implementation Strategy

**CLI Discovery:**
- Use new `ArgparseExtractor` (AST-based, similar to `ComponentExtractor`)
- Detect `argparse.ArgumentParser()`, `add_subparsers()`, `add_parser()` calls
- Extract command names, help text, arguments
- Generate capability IDs: `CAP-CLI-{COMMAND_NAME_UPPER}`

**Gate Discovery:**
- Search for `GATE-\d{3}` pattern in file contents (grep-like)
- Parse docstrings for `GATE-###: Description` format
- Look for `validate_*` function names as secondary signal
- Generate capability IDs: `CAP-GATE-{NUM}-{SANITIZED_NAME}`

**Schema Discovery:**
- Scan `schemas/**/*.json`
- Parse JSON, extract `$id` and `title` fields
- Generate capability IDs: `CAP-SCHEMA-{TITLE_SANITIZED}`

**Conflict Resolution:**
- Apply precedence rule: **code > schemas > templates > docs**
- Map evidence types: `ast=1, schema_id=2, docstring=3, string_match=4`
- Select lowest-precedence source as winner
- Record conflicts in output

## Step 2: File Inventory

**Module:** `file_inventory_builder.py`

### Objective
Build a complete inventory of all repository files with objective metadata:
- File classification (entrypoint, module, schema, config, doc, etc.)
- Python-specific: imports, defines (classes/functions), argparse commands, subprocess calls, path literals
- JSON-specific: parseable status, schema_id, title
- SHA256 hash for each file

```python
class FileInventoryBuilder:

    def _analyze_python_file(self, file_path: Path) -> Dict
```

### File Classification Logic

```python
# Classification enum:
- python_entrypoint  # Has shebang, if __name__=="__main__", or in scripts/
- python_module      # Regular .py file
- python_test        # test_*.py or *_test.py
- schema_json        # JSON with $schema or $id
- template_json      # JSON with 'template' or 'spec' in name
- spec_doc           # .md with 'spec', 'instruction', 'requirement' in name
- doc                # Other .md files
- config             # .yaml, .yml files
- data               # Other JSON files
- unknown            # Everything else
```

### JSON Analysis

```python
def _analyze_json_file(self, file_path: Path) -> Dict:
    try:
        data = json.loads(file_path.read_text(encoding='utf-8'))
        return {
            'json_parseable': True,
            'schema_id': data.get('$id'),
            'title': data.get('title')
        }
    except json.JSONDecodeError:
        return {'json_parseable': False, 'schema_id': None, 'title': None}
```

### Deterministic Ordering

- Walk files in **lexicographic order** (use `sorted()`)
- Sort all extracted lists by line number
- Use `sort_keys=True` for JSON serialization

One record per line (JSON Lines format):


## Step 3: Purpose Registry

**Module:** `purpose_registry_builder.py`

### Objective
Map files to capabilities with evidence-based justifications. Establish "gate scope" (files that MUST be mapped) and apply deterministic heuristics.

### Key Classes

```python
class PurposeRegistryBuilder:
    def __init__(self, capabilities_path, inventory_path, repo_root):
        self.capabilities = self._load_capabilities(capabilities_path)
        self.inventory = self._load_inventory(inventory_path)
        self.file_index = {rec['path']: rec for rec in self.inventory}
        self.cap_index = {cap['capability_id']: cap for cap in self.capabilities['capabilities']}

    def build_registry(self) -> Dict
    def _determine_gate_scope(self) -> Set[str]
    def _map_files_to_capabilities(self, gate_scope) -> List[FileMapping]
    def _enrich_with_call_graph(self, mappings)
    def _validate_registry(self, mappings, gate_scope) -> Dict
```

### Gate Scope Determination

**What MUST be mapped:**
1. All `python_entrypoint` files
2. All `schema_json` files
3. Files referenced with `required_exists_guess=true` in inventory
4. Files listed in `capabilities[].candidate_files`

### Deterministic Mapping Heuristics (In Order)

1. **Schema files** → `CAP-SCHEMA-{TITLE}`
   - High confidence
   - Maps directly to schema capability

2. **Gate scripts** → `CAP-GATE-{NUM}-{NAME}`
   - High confidence
   - Detect by `GATE-###` pattern or `validate_*` function + evidence path pattern

3. **CLI entrypoints** → `CAP-CLI-{SCRIPT_NAME}` + secondary `CAP-CLI-{COMMAND}` per command
   - High confidence
   - Requires `argparse_commands[]` non-empty

4. **Library modules** → `CAP-LIB-SHARED_SUPPORT` or inferred from importers
   - Medium confidence
   - Will be enriched by call graph

5. **Other files** → Best-effort mapping or explicit exclusion
   - Low confidence → added to `review_queue`

### Call Graph Construction

**Purpose:** Populate `called_by_observed[]` from import relationships.

```python
class CallGraphBuilder:
    """Build import graph: module → set of importers"""

    def _build_import_graph(self) -> Dict[str, Set[str]]:
        graph = defaultdict(set)
        for record in self.inventory:
            if record.get('imports'):
                importer_path = record['path']
                for imp in record['imports']:
                    imported_path = self._resolve_import_to_path(imp['module'], importer_path)
                    if imported_path:
                        graph[imported_path].add(importer_path)
        return graph
```

### Registry Validation

**Three critical checks:**

1. **Unmapped gate-scope files** - All gate-scope files MUST have mappings
2. **Unimplemented capabilities** - All `status: implemented` capabilities MUST have at least one file
3. **Duplicate primary implementations** - No capability should have multiple primary files (unless explicitly allowed)

## Main CLI Orchestrator

**File:** `scripts/P_01260207233100000XXX_capability_mapper.py`

### CLI Interface

```bash
python capability_mapper.py [--step {1,2,3,all}] [--repo-root PATH]

Options:
  --step {1,2,3,all}   Run specific step or all (default: all)
  --repo-root PATH     Repository root (default: auto-detect via .git/)

Examples:
  python capability_mapper.py                    # Run all steps
  python capability_mapper.py --step 1           # Just discover capabilities
  python capability_mapper.py --step 3           # Just build registry (requires steps 1+2)
```

### Orchestrator Logic

```python
def main():
    # 1. Find repo root (walk upward until .git/ exists)
    repo_root = find_repo_root(args.repo_root)

    # 2. Setup output directories
    output_root = repo_root / '.state' / 'purpose_mapping'
    evidence_root = repo_root / '.state' / 'evidence' / 'purpose_mapping'

    # 3. Run requested steps
    results = {}
    if '1' in steps_to_run:
        results['step1'] = run_step1(repo_root, output_root, evidence_root)
    if '2' in steps_to_run:
        results['step2'] = run_step2(repo_root, output_root, evidence_root)
    if '3' in steps_to_run:
        results['step3'] = run_step3(repo_root, output_root, evidence_root)

    # 4. Print summary and exit with appropriate code
    all_passed = all(results.values())
    sys.exit(0 if all_passed else 1)
```

### Error Handling Strategy

**Philosophy: Fail-safe with warnings**

- **Step 1/2**: Non-blocking errors for individual file parsing (skip file, record warning)
- **Step 3**: Blocking errors for missing inputs (FAIL immediately if Steps 1/2 outputs missing)
- Low-confidence mappings go to `review_queue` but don't block output
