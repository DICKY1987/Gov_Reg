This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Gov_Reg is a deterministic governance automation framework. The core product is a **Unified SSOT Registry** (148-column JSON) that tracks every file in the repo with governance metadata. Changes flow through a **6-state lifecycle** (PLANNED, PRESENT, DEPRECATED, DELETED, ORPHANED, CONFLICT) enforced by validators, gates, and write policies.

The system enforces compliance through **preventative real-time enforcement**, not just CI checks.

## Architecture

### Data Flow
```
Registry SSOT (148-column JSON)
  ↓ governed by
Write Policy (YAML) + Derivations (safe DSL)
  ↓ enforced by
GEU Layer (SCHEMA/RULE/VALIDATOR/RUNNER/FAILURE_MODE/EVIDENCE/TEST/REPORT)
  ↓ checked at
Gate Layer (Completeness exit 10, Validity exit 11)
  ↓ triggered by
File System Layer (watchers, event emission, identity resolution)
```

### Registry Transition System (`src/registry_transition/`)

The core package implementing the 6-state lifecycle. This is the most tested code (90% coverage, 26 tests):

| Module | Purpose |
|--------|---------|
| `lifecycle_state.py` | Enums + dataclasses for the 6 states |
| `state_machine.py` | Contract loader; validates transitions against `transition_contract.bundle.yaml` |
| `identity_resolver.py` | 4-step file matching: exact path → rename intent → strong identity (20-digit file_id) → orphan/conflict |
| `field_precedence.py` | Classifies fields as authoritative (must overwrite), inferred (preserve planned), or immutable (never change) |
| `registry_writer.py` | Atomic write protocol (temp → validate → rename) with automatic backups |
| `gates.py` | Completeness + validity gate checks |

### GEU System (`src/P_01999000042260124484_geu_governance/`)

9 Python modules for Governance Enforcement Units. Each GEU bundles: schema, rule, validator, runner, failure_mode, evidence, test, report. Key modules: `validation.py` (9 rules), `generate.py` (GEU generation), `cli.py` (CLI interface).

### Repo Auto-Ops (`src/repo_autoops/`)

Automation descriptor system (20+ modules): file watcher daemon, reconciler, audit logger, work queue, identity pipeline, classifier, and write policy validator.

### Path Registry (`PATH_FILES/path_registry.py`)

Singleton-based semantic path resolver. Maps logical keys (e.g., `REGISTRY_UNIFIED`) to filesystem paths via `PATH_FILES/config/path_index.yaml` (80+ entries). Enforces repo-root boundaries and tracks access history.

## Key Conventions

### File ID Format
**Mandatory 20-digit prefix**: `01999000XXXXXXXXXX`
- Python modules: `P_01999000XXXXXXXXXX_name.py`
- Data files: `01999000XXXXXXXXXX_name.ext`
- Duplicate IDs = Critical violation

### Registry Write Protocol
**Single-writer semantics**: `Operation → Validate → Atomic write (temp → rename) → Backup`
- All writes validate against `UNIFIED_SSOT_REGISTRY_WRITE_POLICY.yaml`
- Computed fields use derivation formulas (safe DSL only, no `eval()`)
- Backups stored in `REGISTRY/backups/` before destructive changes

### Path Safety
- Use `resolve_path(KEY)` or `resolve_module_path(KEY)` from `path_registry.py`
- No hardcoded absolute paths; key format: `UPPERCASE_WITH_UNDERSCORES` or `namespace:key`

### Field Precedence (for registry updates)
- **Authoritative** (must overwrite): `size_bytes`, `sha256`, `mtime_utc`, `last_seen_utc`
- **Inferred** (preserve planned value): `artifact_kind`, `layer`, `language`, `framework`
- **Immutable** (never change): `file_id`, `record_id`, `created_utc`, `original_doc_id`

### GEU Roles
When adding files to the registry, classify by `geu_role`:
`SCHEMA` | `RULE` | `VALIDATOR` | `RUNNER` | `FAILURE_MODE` | `EVIDENCE` | `TEST` | `REPORT`

### Code Style
- PEP 8, 4-space indent, 100-char line length
- snake_case for functions/variables, PascalCase for classes
- Type hints required for public APIs (mypy strict mode)
- Google-style docstrings

## Important Constraints

- Never manually edit `*.phase_contract.json` files (use RFC-6902 patches)
- Never hardcode absolute file paths (use `path_registry.py`)
- Never use `eval()` in derivations (only the safe DSL)
- Never create duplicate 20-digit IDs
- Never write to registry without validating against write policy
- Always validate path registry before operations
- Always back up registry before destructive changes
- Always run tests after schema/validator changes

## CI/CD

`.github/workflows/evidence_integrity.yml` enforces on push/PR:
1. `coverage.json` must contain pytest-cov metadata (`.meta.version`)
2. `junit.xml` must contain pytest signature
3. `determinism_score.json` must have `metrics_lines_read > 0`
4. All 26 tests must pass
5. Coverage must be >= 90%

Evidence files live in `.state/evidence/final/` and must be tool-generated, never hand-written.
