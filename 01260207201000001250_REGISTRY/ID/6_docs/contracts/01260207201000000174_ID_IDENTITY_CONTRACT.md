## Purpose

This document defines the **single source of truth** for ID terminology, preventing semantic drift and enabling correct implementation of the canonicality system.

**Problem it solves:** The terms `file_id`, `doc_id`, and "20-digit" vs "22-digit" were used inconsistently, causing implementation conflicts.

---

## Core Definitions (Immutable)

### `file_id`
**Definition:** 20-digit numeric string (no prefix, no underscore)  
**Format:** `\d{20}`  
**Example:** `01999000042260124027`  
**Used for:** Unique identifier for any file in the system  
**Storage:** Numeric string in registry, inventory, databases

**RULES:**
- ✅ VALID: `01999000042260124027`
- ❌ INVALID: `P_01999000042260124027` (has prefix)
- ❌ INVALID: `01999000042260124027_` (has underscore)
- ❌ INVALID: `0199900004226012402` (19 digits)
- ❌ INVALID: `019990000422601240270` (21 digits)

### `doc_id`
**Definition:** Python/document identifier with `P_` prefix + 20-digit `file_id`  
**Format:** `P_\d{20}`  
**Example:** `P_01999000042260124027`  
**Used for:** Python module identifiers, documentation artifacts  
**Storage:** String in code, docs, but stored as `file_id` in registry

**RULES:**
- ✅ VALID: `P_01999000042260124027`
- ❌ INVALID: `01999000042260124027` (missing prefix)
- ❌ INVALID: `P_01999000042260124027_` (has trailing underscore)
- ❌ INVALID: `D_01999000042260124027` (wrong prefix)

**Relationship to `file_id`:**
```python
doc_id = f"P_{file_id}"
file_id = doc_id.removeprefix("P_")
```

### `filename_id_prefix`
**Definition:** ID prefix as it appears in filenames (with underscore boundary)  
**Format:** `^(?:P_)?\d{20}_`  
**Examples:**
- Python file: `P_01999000042260124027_id_allocator.py`
- Regular file: `01999000042260124027_registry.json`

**RULES:**
- The underscore `_` is a **boundary marker** (separates ID from name)
- Python files: `P_{file_id}_<name>`
- Other files: `{file_id}_<name>`
- The underscore is **not part of the ID itself**

### `enhanced_id` (22-digit system)
**Definition:** 22-digit identifier from COUNTER_STORE with scope prefix  
**Format:** `\d{22}`  
**Example:** `0199900004226012400001`  
**Used for:** Unified allocator with scope/counter segments

**RULES:**
- ✅ This is a **different identifier type** from `file_id`
- ❌ Do NOT call this `file_id` (causes confusion)
- Alternative names: `allocation_id`, `reservation_id`, `ledger_id`
- Structure: `<prefix:15><counter:5>` (05d format)

## Digit Count Policy

| ID Type | Digits | Prefix | Boundary | Example |
|---------|--------|--------|----------|---------|
| `file_id` | 20 | None | None | `01999000042260124027` |
| `doc_id` | 20 | `P_` | None | `P_01999000042260124027` |
| `filename_id_prefix` (Python) | 20 | `P_` | `_` | `P_01999000042260124027_` |
| `filename_id_prefix` (other) | 20 | None | `_` | `01999000042260124027_` |
| `enhanced_id` | 22 | None | None | `0199900004226012400001` |

## Canonical Regex Patterns

**Import these from:** `govreg_core.P_01999000042260125002_canonical_id_patterns`

```python
# DO NOT redefine these patterns elsewhere
FILE_ID_PATTERN = re.compile(r'^\d{20}$')
DOC_ID_PATTERN = re.compile(r'^P_\d{20}$')
FILENAME_PREFIX_PATTERN = re.compile(r'^(?:(?P<doc_prefix>P_))?(?P<file_id>\d{20})_')
ENHANCED_ID_PATTERN = re.compile(r'^\d{22}$')
```

## Extraction Rules

### From filename to `file_id`:
```python
# Input: "P_01999000042260124027_id_allocator.py"
# Output: "01999000042260124027"

match = FILENAME_PREFIX_PATTERN.match(filename)
if match:
    file_id = match.group('file_id')  # Always 20 digits, no prefix
```

### From `file_id` to `doc_id` (Python files):
```python
# Input: "01999000042260124027"
# Output: "P_01999000042260124027"

doc_id = f"P_{file_id}"
```

### From `doc_id` to `file_id`:
```python
# Input: "P_01999000042260124027"
# Output: "01999000042260124027"

file_id = doc_id.removeprefix("P_")
```

```jsonl

  "doc_id": "P_01999000042260124027",
  "path": "scripts/P_01999000042260124027_id_allocator.py",
  "filename_prefix_kind": "DOC_PREFIXED",
  "canonicality": "CANONICAL"
}
```

### In Registry (governance_registry_unified.json):
```json
{
  "file_id": "01999000042260124027",
  "relative_path": "scripts/P_01999000042260124027_id_allocator.py",
  "canonicality": "CANONICAL"
}
```

**RULE:** `file_id` field always stores 20-digit numeric string (no prefix)

## Migration Path

### Legacy System (ID_COUNTER) → New System (COUNTER_STORE)

| System | ID Type | Digits | Counter Source |
|--------|---------|--------|----------------|
| Legacy | `file_id` | 20 | `ID_COUNTER.json` |
| New | `enhanced_id` | 22 | `COUNTER_STORE.json` |
| Coexistence | Both | 20 & 22 | Both counters |

**Policy:**
- Both systems operational during transition
- `file_id` (20-digit) remains the primary identifier
- `enhanced_id` (22-digit) is for new allocations with scope
- No field should use both names interchangeably

## Validation Rules

### Required Checks:

1. **Prefix Detection:**
   ```python
   from govreg_core.P_01999000042260125002_canonical_id_patterns import FILENAME_PREFIX_PATTERN
   
   match = FILENAME_PREFIX_PATTERN.match(filename)
   if match:
       has_doc_prefix = bool(match.group('doc_prefix'))
       file_id = match.group('file_id')
   ```

2. **Uniqueness:**
   ```python
   # Check against registry before accepting
   if file_id in registry:
       raise ValueError(f"Duplicate file_id: {file_id}")
   ```

3. **Format Validation:**
   ```python
   from govreg_core.P_01999000042260125002_canonical_id_patterns import validate_id
   
   if not validate_id(value, "FILE_ID"):
       raise ValueError(f"Invalid file_id format: {value}")
   ```

## Anti-Patterns (Forbidden)

### ❌ DO NOT:

1. **Mix prefix in `file_id` field:**
   ```python
   # WRONG
   {"file_id": "P_01999000042260124027"}
   
   # CORRECT
   {"file_id": "01999000042260124027", "doc_id": "P_01999000042260124027"}
   ```

2. **Define local regex patterns:**
   ```python
   # WRONG
   FILE_ID_RE = re.compile(r'^\d{20}_')
   
   # CORRECT
   from govreg_core.P_01999000042260125002_canonical_id_patterns import FILE_ID_PATTERN
   ```

3. **Call 22-digit values `file_id`:**
   ```python
   # WRONG
   file_id = "0199900004226012400001"  # 22 digits
   
   # CORRECT
   enhanced_id = "0199900004226012400001"
   file_id = "01999000042260124027"  # 20 digits
   ```

4. **Include underscore in ID value:**
   ```python
   # WRONG
   file_id = "01999000042260124027_"
   
   # CORRECT
   file_id = "01999000042260124027"
   filename = f"{file_id}_allocator.py"
   ```
