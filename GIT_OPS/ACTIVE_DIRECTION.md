# GIT_OPS Active Direction

## Current decision

`GIT_OPS` is now organized around a GitHub-native-first planning model.

- GitHub Issues and Projects are the active planning and execution surface.
- Native GitHub capabilities, templates, fields, views, milestones, sub-issues, and light Actions automation are the preferred design direction.
- Archived custom control-plane, projection, CLI, and reconciliation designs are retained for history only and must not be treated as active requirements.
