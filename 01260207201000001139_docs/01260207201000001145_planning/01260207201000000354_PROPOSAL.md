# Problem
The governance registry has mismatches between schema, data, policies, roles, and documentation. The audit identified conflicts that can cause validation failures, stale logic, and onboarding confusion.

# Goals
- Align schema v4 with unified registry data and dictionary.
- Remove doc_id references from live artifacts.
- Reconcile write policy and derivation definitions.
- Align bundle_role and geu_role taxonomy with actual usage.
- Enforce validation in CI (schema + no-doc-id).
- Normalize ID naming for registry-tracked artifacts.
- Synchronize documentation with current behavior.

# Non-Goals
- Refactor unrelated LP_LONG_PLAN artifacts.
- Rewrite historical evidence or archived reports.
- Change external tools outside this repository.

# Approach
1. Define validation gates and evidence artifacts.
2. Update schema, registry data, and dictionary in parallel with doc cleanup and policy alignment.
3. Align role taxonomy and integrate CI gates.
4. Normalize naming for registry-tracked artifacts based on scanner report.
5. Update tests to cover new validation rules.
6. Reconcile goals to evidence.

# Deliverables
- Schema v4 file and updated unified registry JSON.
- Updated column dictionary and ingest spec.
- Updated write policy and derivations.
- Updated bundle role documentation and SSOT definitions.
- CI workflow with schema and no-doc-id gates.
- ID format scan report and updated references.
- Updated tests and documentation.

# Risks
- Schema changes may require wider updates to downstream tooling.
- Naming normalization can break references if not fully updated.
- Policy/derivation changes may surface hidden inconsistencies.

# Open Questions
- Should legacy doc_id references be preserved in a dedicated legacy appendix?
- Which registry-tracked files are allowed to retain DOC- prefixes during transition?
