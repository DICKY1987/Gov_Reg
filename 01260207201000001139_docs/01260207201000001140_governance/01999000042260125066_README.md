# Registry Documentation Templates

This directory contains standardized templates for registry documentation across all identifier registries in the GOVERNANCE system.

## Templates

- **TEMPLATE_REGISTRY_README.md** - Standard registry overview and structure
- **TEMPLATE_REGISTRY_RUNBOOK.md** - Troubleshooting and emergency procedures
- **TEMPLATE_REGISTRY_USAGE.md** - Quick start and integration guide

## Usage

### For Registry Maintainers

When creating or updating registry documentation:

1. Copy the appropriate template
2. Replace all `{{PLACEHOLDER}}` variables with registry-specific values
3. Save to your registry directory (conflicts/, gates/, policies/, precedents/, rules/)

### Placeholder Reference

Common placeholders used in templates:

| Placeholder | Description | Example |
|-------------|-------------|---------|
| `{{REGISTRY_NAME}}` | Human-readable registry name | "Conflict", "Quality Gate", "Policy" |
| `{{type_id}}` | Technical type identifier | "conflict_id", "gate_id", "policy_id" |
| `{{classification}}` | ID classification | "minted", "derived", "composite" |
| `{{owner}}` | Responsible team/system | "Conflict Resolution", "CI/CD Team" |
| `{{description}}` | Brief registry purpose | "Stable identifier for known system conflicts" |
| `{{format_pattern}}` | ID format pattern | "CONFLICT-{SEQ}", "GATE-{PHASE}-{NAME}-{SEQ}" |
| `{{format_regex}}` | Validation regex | `^CONFLICT-([0-9]{3,})$` |
| `{{format_example}}` | Example ID format | "CONFLICT-<SEQ>", "GATE-<PHASE>-<NAME>-<SEQ>" |
| `{{categories_section}}` | Categories list or "*No categories defined*" | See examples below |
| `{{automation_section}}` | Automation info or "*No automation configured*" | Custom per registry |
| `{{registry_dir}}` | Registry subdirectory name | "conflicts", "gates", "policies" |
| `{{REGISTRY_FILE}}` | Registry filename (without .yaml) | "CONFLICT_REGISTRY", "GATE_REGISTRY" |
| `{{example_category}}` | Example category for usage docs | "DOC", "SECURITY", "PRE_COMMIT" |
| `{{YYYY-MM-DD}}` | Last updated date | "2026-01-09" |

### Category Section Examples

**No categories:**
```markdown
*No categories defined*
```

**With categories:**
```markdown
- **SECURITY** - SECURITY category
- **QUALITY** - QUALITY category
- **PROCESS** - PROCESS category
- **COMPLIANCE** - COMPLIANCE category
```

## Active Registries

The following registries use these templates:

1. **conflicts/** - Conflict Identifier Registry (`conflict_id`)
2. **gates/** - Quality Gate Identifier Registry (`gate_id`)
3. **policies/** - Policy Identifier Registry (`policy_id`)
4. **precedents/** - Precedent Identifier Registry (`precedent_id`)
5. **rules/** - Rule Identifier Registry (`rule_id`)

## Benefits

- **Consistency** - All registries follow the same documentation structure
- **Maintainability** - Updates to templates propagate to all registries
- **Onboarding** - New registry creators have clear patterns to follow
- **Quality** - Reduces documentation drift and incomplete sections

## Updating Templates

When updating templates:

1. Update the template file in this directory
2. Document changes in this README
3. Notify registry maintainers of changes
4. Update affected registry documentation as needed
